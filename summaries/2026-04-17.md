🛡️ **PlanGuard: Defending Agents against Indirect Prompt Injection via Planning-based Consistency Verification** [source](http://arxiv.org/pdf/2604.10134v1.pdf) #security `7/10` 

 *Guangyu Gong, Zizhuang Deng (Shandong University, Suzhou Research Institute of Shandong University)*

 Separating planning from retrieval and verifying every tool call against a clean reference plan achieves effectively zero tool-execution hijacks with minimal usability loss.
 - Planning-based context isolation that derives an allowlist of valid tool actions from only the trusted user instruction can drive indirect prompt injection Attack Success Rate (ASR) on InjecAgent from 72.8% to 0% while keeping False Positive Rate (FPR) at 1.49%.
 - A two-stage verifier that blocks unauthorized tool invocations with hard constraints and then uses an intent-level check for parameter deviations cuts FPR from 27.0%→0.97% (Direct Harm) and 38.01%→3.28% (Data Stealing) without sacrificing the 0% ASR result.
 - Even when the underlying agent is made deliberately over-compliant (“execute commands found in tool returns”), isolating the planner from retrieved content turns IPI from a probabilistic alignment problem into a deterministic execution firewall, shifting security guarantees to the action layer.

🧩 **Challenges and Future Directions in Agentic Reverse Engineering Systems** [source](http://arxiv.org/pdf/2604.14317v1.pdf) #cyber `6/10` 

 *Salem Radey, Jack West, Kassem Fawaz (University of Wisconsin-Madison)*

 Agentic reverse engineering is constrained less by tool availability than by obfuscation brittleness, context-window economics, and the safety gap introduced when LLMs are allowed to execute powerful analysis commands autonomously.
 - Common obfuscations can collapse LLM-based decompilation effectiveness—Control Flow Flattening (CFF) and Bogus Control Flow (BCF) each reduce success rates by over 70%—implying agentic RE systems will systematically fail on realistic closed-source targets unless deobfuscation is built into the pipeline.
 - Token limits are a first-order bottleneck for static RE at scale, with binary-diff summarization experiments requiring roughly 100M–500M tokens per model across 104 binaries, making naive “dump decompiler output into the LLM” approaches cost-prohibitive and coverage-limiting without aggressive context reduction.
 - Many hybrid/dynamic agent implementations permit near-unrestricted shell or debugger command execution, creating a plausible path for adversarial binaries to induce unsafe tool calls and host compromise, so dynamic-analysis guardrails and verification layers become mandatory rather than optional.
